# 实例：生产者与消费者模型


 生产者 | 消息类 | 消费者
  ------|--------|-------
 Producer | Message | Consumer
 
 ~~~java
 public class PandCmodel {
    public static void main(String[] args) {
        Message msg = new Message();
        new Thread(new Producer(msg)).start(); //启动生产者线程
        new Thread(new Consumer(msg)).start(); //启动消费者线程
    }
}

class Producer implements Runnable {
    private Message msg;

    public Producer(Message msg) {
        this.msg = msg;
    }

    @Override
    public void run() {
        for (int i = 0; i < 100; i++) {
            if (i % 2 == 0) {
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                this.msg.setTitle("孙涛");
                this.msg.setContent("大帅哥");
            } else {
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                this.msg.setTitle("罗文刚");
                this.msg.setContent("猥琐第一人");
            }
        }
    }
}

class Consumer implements Runnable {
    private Message msg;

    public Consumer(Message msg) {
        this.msg = msg;
    }

    @Override
    public void run() {
        for (int j = 0; j < 100; j++) {
            try {
                Thread.sleep(10);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(this.msg.getContent() + "  -  " + this.msg.getTitle());
        }
    }
}

class Message {
    private String title;
    private String content;

    public void setContent(String Content) {
        this.content = content;
    }

    public void setTitle(String title) {
        this.title = title;
    }

    public String getContent() {
        return content;
    }

    public String getTitle() {
        return title;
    }
}
 ~~~
 
 通过整个代码执行，有两个主要问题：
 
 * 数据不同步
 * 部分生产、消费结果有重复生产和重复取出的现象
 
 > 问题就是没有考虑同步问题导致的
 
 如果要解决问题，首先考虑就是怎么解决数据同步；
 
## 方法1:synconized 关键字
 
 引入`synchronized`关键字，并重写`get()`和`set()`方法

 public class PandCmodel {

 同步处理的时候肯定需要一个**同步的对象**，这个对象要交由Message类处理是最合适的。
 
> 至少数据保持一致了，但是**重复操作**问题依然存在
 
## 线程等待与唤醒

在Message中设置一个信标
 
* Message接受生产者生产一条消息
* 信标变换，现在允许消费者使用
* 消费者使用掉这个信息
* 信标变换，现在允许等待生产者生产
* Message允许接受生产者生产下一条信息

主要依靠Object类中的方法实现

~~~java
public final void wait() throws Exception; //死等，会抛出异常待处理
public final void wait(long timeout, int nanos) throws InterruptedException; //等一段时间，会抛出中断异常

public final void notify(); //唤醒第一个等待线程，其他的还在等
public final void notifyall(); //唤醒全部等待线程，谁优先级高谁先执行
~~~

因此对Message类进行修改

~~~java
//修改后的Message类如下，这就是多线程最原始的处理方案

class Message {
    private String title;
    private String content;
    private boolean flag = true;// true只允许生产，false只允许消费

    public synchronized void set(String title, String content) {
        if (this.flag == false) {
            try {
                super.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        this.title = title;
        this.content = content;
        this.flag = false;//已经生产过了
        super.notify();//唤醒等待的线程
    }
    public synchronized String get() {
        if (this.flag == true) {
            try {
                super.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        try {
            Thread.sleep(10);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        try {
            return this.title + "  -  " + this.content;
        } finally {//无论如何都必须要执行
            this.flag = true;
            super.notify();
        }
    }
}

~~~
 
这样就实现用原生代码实现控制多线程操作了；
